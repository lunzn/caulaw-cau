# Impeccable — 项目设计上下文

本文件由 `/impeccable teach` 建立并已由作者补全，供后续界面与视觉相关工作时引用。

---

## Design Context

### Users

- **产品定位（来自 `app/layout.tsx` metadata）**：中国农业大学校园智能助手管理平台（caulaw）。
- **使用者**：**学生与教职工**（两类兼有）。
- **使用情境**：已登录用户在 Web 控制台管理「微信 Bot 连接状态、扫码登录、定时任务（cron）」等；界面语言为中文（`lang="zh-CN"`）。

### Brand Personality

- **三字调性**：**克制、极简**（在此基础上可叠加「清晰、可信」等工具型情绪，但视觉表达仍以克制为先）。
- **色彩策略（已定）**：**绿色**为主品牌 / 主操作色；**黄色**为辅助色（强调次要信息、提示、次要按钮或与主色搭配的点缀），两处均需满足对比度与可读性。
- **情绪目标**：界面不显嘈杂；信息层级靠留白、字号与少色层次完成，而非堆装饰。

### Aesthetic Direction

- **主题**：**跟随系统**（与现有 `next-themes` `defaultTheme="system"` 一致），勿默认锁死浅色或深色。
- **技术基线**：Next.js App Router、Tailwind v4、shadcn 风格组件；`globals.css` 使用 OKLCH 语义色与 light/dark 变量。**落色时**将品牌绿/黄写入 `primary`、辅助语义（如 `accent` 或专用令牌），并在明暗两套 `:root` / `.dark` 下各调一版，避免纯灰主色或与背景粘连。
- **字型**：当前为 Geist / Geist Mono。若后续替换，选「中性、易读、略偏学术/工具」的配对，仍保持极简，避免花哨 display。

### Design Principles

1. **中文优先**：文案、错误提示、空状态说明均以自然中文书写；英混仅保留必要术语（如 cron、WeChat userId）。
2. **克制与极简**：少组件皮肤变种；减少描边/阴影种类；用间距与对齐建立秩序，避免侧条高亮、渐变字等 Impeccable 明确禁止的套路。
3. **可信与可恢复**：涉断开连接、删除凭据、清空任务等须有清晰后果与确认；Bot 状态（在线/扫码/错误）一目了然。
4. **绿色主、黄色辅**：主 CTA、关键状态与品牌触点用绿；黄用于辅助强调，避免两种颜色平均铺满界面（约 60% 中性表面 / 30% 文字与界 / 10% 色块重心）。
5. **可访问性底线**：表单与按钮保留可见 focus、语义与对比度；动效尊重 `prefers-reduced-motion`。
6. **色彩实现**：延续 OKLCH；中性基底可向绿色相轻微染色以保持成套感，高光与黄色辅色在极浅/极深背景下略降 chroma，避免刺眼。
